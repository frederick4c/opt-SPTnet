# Submission guide (examiner-facing)

This file describes the layout of the GitLab submission repository and how to
reproduce the results in the report. The submission repository contains the
software package, the thesis report sources and PDF, and the small
reproducibility artifacts that the report figures and tables are built from.
Large raw data and model weights are not committed; their locations are noted
below.

## Repository layout

| Path | Contents |
| --- | --- |
| `src/sptnet/` | The installable Python package (models, data, training, inference, segmentation, visualization). |
| `tests/` | The pytest suite. |
| `docs/` | Sphinx documentation sources. |
| `report/` | Thesis report and executive summary: LaTeX sources, `main.bbl`, figures, `main.pdf`, and `executive_summary.pdf`. |
| `notebooks/` | Analysis and figure-generation scripts/notebooks referenced below. |
| `experiments/benchmarks/` | Runtime benchmark harness, per-run timings, and the analyzed `results/` summaries. |
| `acc_eval/` | Short-run numerical-acceleration neutrality check used for `amp_neutrality.pdf`. |
| `diff_evals/` | Matched per-track evaluation outputs (small CSVs). Raw inference `*.h5` are excluded. |
| `RealData/` | Selected stitched-track CSVs and segmentation manifests only. Raw movies, tiles, and per-tile inference files are excluded. |
| `slurm/` | Cluster job scripts used to produce the runs. |

## Building the report

```bash
cd report
latexmk -pdf main.tex      # or: pdflatex main && biber main && pdflatex main x2
```

The committed `report/main.pdf` is the reference build.

## Reproducing the software results

Install and test the package (CPU is sufficient for the test suite):

```bash
python -m pip install -e ".[test]" -c constraints.txt
pytest
```

See `README.md` for the full CLI workflow (generate -> train -> infer -> segment
-> stitch) and `CONTRIBUTING.md` for the developer workflow.

## Figure and table provenance

Each report figure/table traces to a committed artifact or a script:

| Report item | Source artifact / script |
| --- | --- |
| Runtime speedup table + per-epoch figure (`benchmark_per_epoch_train.pdf`) | `experiments/benchmarks/results/summary_*.{csv,md}`; per-run `epoch_timing.csv` under the benchmark run dirs; `experiments/benchmarks/analyze_benchmarks.py`. |
| Numerical-acceleration neutrality (`amp_neutrality.pdf`) | `acc_eval/results/loss_curves.csv`; regenerated by `acc_eval/make_amp_neutrality_fig.py` after `acc_eval/analyze_acc_eval.py`. |
| Reproduction bias (`reproduction_bias.pdf`), transfer cost (`transfer_cost.pdf`), diffusion variance (`diffusion_variance.pdf`) | Matched per-track CSVs under `diff_evals/**/*_matched_*.csv`; regenerated by `notebooks/reeval_matched_fixed.py`. |
| Reproduction-agreement and diffusion-transfer tables | `diff_evals/final/comparison_plots/*_matched_ranking.csv` and `diff_evals/forget/*_matched_ranking.csv`. |
| Pipeline overview (`pipeline_overview.pdf`), data-generation comparison (`data_generation_comparison.pdf`) | TikZ source `report/figures/pipeline_overview.tex`; `notebooks/figs.ipynb`. |
| Real-data figures (`ft_success.png`, `ft_fail.png`, `realdata_overdetection.pdf`, `pre_ft_real.png`) | Selected stitched-track CSVs and segmentation manifests under `RealData/`; raw movies and per-tile inference outputs are excluded (see below). |

The earlier `loss_vs_walltime.pdf` analysis was cut from the final report because
it compared different training objectives. The script is not needed for the
submitted figures.

## Excluded large files and where they live

The following are intentionally not committed (size). They were produced on the
CSD3 cluster and the project working tree:

- Synthetic training data (`TrainData/`, generated `*.h5`) — regenerate with
  `sptnet-generate-training-data` (deterministic given the seed).
- Diffusion-condition and forget-set inference outputs (`diff_evals/**/*.h5`) —
  regenerate with `sptnet-inference`; the small matched-eval CSVs are committed.
- Real microscopy movies (`RealData/**/*.h5`, `RealData/**/*.tif`, videos) and
  per-tile inference outputs — only stitched-track CSVs and segmentation
  manifests are committed.
- Trained model weights (`*.pt`, `trained_model`) — available from the author on
  request.

## The two release remotes

This project is maintained against two remotes with different ignore profiles:

- **Public GitHub** (code only): uses `.gitignore`. Excludes the report,
  notebooks, raw data, and internal `context/` notes.
- **GitLab submission** (this repo): uses `.gitignore.submission`. Adds the
  report sources/PDF, notebooks, and small reproducibility artifacts.

To prepare the submission checkout, use the submission ignore profile:

```bash
cp .gitignore.submission .gitignore
git add -A && git commit -m "Submission: code + report + artifacts"
git push gitlab <branch>
```
